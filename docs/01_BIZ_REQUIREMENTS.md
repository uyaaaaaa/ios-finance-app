# ビジネス要件定義書

## 1. プロジェクトの目的

**「家計管理のハードルを極限まで下げ、継続の習慣化を実現する」**
家計簿を付けるという行為における「アプリを探して開く」「金額を入力する」といった摩擦係数を最小化し、ユーザーが自身の収支状況（予算との乖離）を常に把握できる環境を構築する。

## 2. ターゲットユーザー

- **属性：** 家計簿アプリをインストールしたものの、入力の面倒さから数日で放置してしまった経験のある iOS ユーザー。
- **課題：** \* レシートを溜めてしまい、後からまとめて入力するのが苦痛。
  - 多機能すぎて、どこを見れば現状がわかるのか直感的に理解できない。

## 3. サービスコンセプト

**「家計簿の最初の一歩を、世界一低くするアプリ」**
入力は iOS のシステム（ウィジェット/ショートカット）に溶け込ませ、アプリ本体は「今月あといくら使えるか」の確認に特化する。

## 4. コアバリュー（提供価値）

- **究極のアクセシビリティ：** アプリを起動する手間すら省く。iOS ウィジェットやショートカットを活用し、ロック画面やホーム画面から瞬時に入力を完了させる。
- **視覚的な現在地把握：** 複雑な分析は排除し、「予算（目標値）と実績の乖離」を最優先で表示。今、自分が安全圏にいるのか、危機的状況なのかを 1 秒で判断させる。

## 5. 主要機能（MVP スコープ）

| 機能カテゴリ     | 機能詳細                                                         | 備考                       |
| :--------------- | :--------------------------------------------------------------- | :------------------------- |
| **超速入力**     | iOS ウィジェット、ショートカット、ロック画面連携による支出入力。 | アプリ本体を開かずに完結。 |
| **予算管理**     | 当月の支出上限（目標金額）の設定。                               | シンプルな数値設定。       |
| **シンプル集計** | 当月の総支出額、目標値に対する残金、進捗率の表示。               | 訴求軸は「目標との乖離」。 |
| **履歴表示**     | 記録した支出の一覧表示、削除、簡易修正。                         | 入力ミス時のバックアップ。 |

## 6. 成功指標 (KPI)

- **入力スピード：** ホーム画面の操作開始から、入力完了通知が出るまで**2 タップ以内**。
- **把握スピード：** アプリ起動後、**1 秒以内**に今月の予算残高が視覚的に理解できること。

## 7. ロードマップ

1. **Phase 1（今回）：** 個人端末へのビルド。コアとなる入力体験と予算管理のプロトタイプ完成。
2. **Phase 2：** TestFlight 等を通じた公開、外部ユーザーからのフィードバック収集。
3. **Phase 3：** 継続性をさらに高めるための通知設計や、UI のブラッシュアップ。
