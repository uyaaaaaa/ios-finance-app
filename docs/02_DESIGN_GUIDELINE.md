# デザインガイドライン

## 1. デザイン・フィロソフィー

**「Tool as an Instinct（本能として使える道具）」**
ユーザーに「家計簿を付けている」という意識をさせないほど生活に溶け込み、かつ現状を冷徹に、しかし分かりやすく提示する「静かな道具」を目指す。

## 2. デザイン決定のための「3 大判断軸」

デザイナーが UI やインタラクションで迷った際は、以下の優先順位に従って決定する。

1. **入力速度 ＞ 入力精度**
   - 入力完了画面での「確認」や「保存ボタン」は排除する。
   - 誤入力の防止にコストをかけるより、誤入力した後に「いかに簡単に直せるか（履歴画面）」にコストをかける。
2. **道具的信頼感 ＞ 情緒的演出**
   - キャラクターや過度な励まし、装飾的なアニメーションは不要。
   - Apple 純正アプリのような、清潔感があり、機能が予測可能な UI を優先する。
3. **直感的現在地 ＞ 詳細な分析**
   - 「何にいくら使ったか」の内訳表示よりも、「予算まであと何円か」という距離感を強調する。

## 3. UI 構造とナビゲーション

画面遷移は最小限とし、以下の 2 階層を基本とする。

- **メイン画面（Dashboard）：**
  - 中央に「今月の残り予算」を巨大な数字で配置。
  - その下に「現在の総支出」と「予算進捗（プログレスバー）」を配置。
- **履歴画面（Log）：**
  - メイン画面を上スワイプ、またはボタンから遷移。
  - 時系列でのリスト。1 タップで削除・修正が可能。

## 4. インプット UX の定義

本アプリの核心部であり、以下の仕様を厳守する。

- **ウィジェット/ショートカット入力：**
  - ウィジェット上のカテゴリボタン（例：食費、日用品、その他）をタップ → 金額入力（テンキー） → 完了。
  - 金額入力後の「確定」動作は、テンキーの「Done」または入力完了後の自動遷移とし、確認ダイアログは一切出さない。
- **カテゴリ設計：**
  - 最大 3〜4 つに絞り、ユーザーが「選ぶ」ことに迷う時間を排除する。

## 5. ビジュアル・スタンダード

- **カラー：** モノトーン（白・黒・グレー）＋ 1 色のアクセント（予算超過時のみ警告色）。
- **フォント：** SF Pro (iOS 標準)。数値の可読性を最大化する。
- **余白：** 広めのホワイトスペース。情報密度を下げ、一点（予算残高）に集中させる。
- **フィードバック：** Taptic Engine を活用し、入力成功時を触覚で伝える。

## 6. アンチパターン（やってはいけないこと）

- **グラフの多用：** 円グラフや折れ線グラフは初期フェーズでは採用しない。
- **チュートリアルの強制：** 直感的に触れる UI を徹底し、説明を不要にする。
- **広告表示・バナー：** 道具としての信頼感を損なう配置は禁止。
